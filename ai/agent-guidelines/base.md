# Repository Guidelines

Use scripts in the `package.json` file for linting, running tests, formatting, and other tasks whenever possible.

## Principles

- DRY: Don't Repeat Yourself
- KISS: Keep It Simple, Stupid
- YAGNI: You Aren't Gonna Need It
- SOLID: Single Responsibility, Open/Closed, Liskov Substitution, Interface Segregation, Dependency Inversion

## Comment Guidelines

All comments must add value for developers working on the codebase. DO NOT add comments documenting the removal of code.
